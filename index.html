Dataset & Data Processing

VideoBadminton Dataset

After searching platforms like Kaggle for labeled badminton clips without success, I discovered the VideoBadminton Dataset containing thousands of clips. However, the dataset presented several challenges of its own:

  • Variable frame counts (1-100+ frames per clip)
  • Only filmed from one angle

The end goal is to be able to analyze videos from different angles, to make it easy for any amateur badminton player to analyze their own gameplay. Currently, it's unknown whether the current dataset will be sufficient to train a model to do this.

Solutions & Architecture

Data Standardization

To address variable frame counts, I implemented a solution to extract exactly 6 frames from each clip, discarding any that didn't meet this threshold.

Pose-Based Processing

Instead of working with raw 1280x720px frames, I use pose estimation models (OpenPose/MediaPipe) to extract body keypoints. This dramatically reduces input size while preserving essential spatial and motion features.

Player Identification

To solve the multiple player challenge, I'm building an auxiliary model that analyzes shuttle trajectory and timing to determine the active player. The main classifier then uses pose data from only that player.

Shuttlecock Tracking with YOLOv8

Tracking the shuttlecock's movement poses unique challenges:

  • Extremely small object size
  • Very high speeds, especially during smashes or drives
  • Frequent motion blur and rapid changes in trajectory

Why YOLOv8?

  • Speed & accuracy: Optimized for real-time detection, crucial for fast-paced badminton footage
  • Small object detection: Improved architecture (anchor boxes, feature aggregation) excels at finding tiny, fast-moving objects
  • Customizable & lightweight: Easily fine-tuned on shuttlecock-specific datasets

Advanced tracking: YOLOv8 integrates with ByteTrack/DeepSORT for temporal consistency, following the shuttle even through brief occlusions or blur.

Challenges introduced by YOLOv8:

  • Requires a large, well-annotated dataset of shuttlecock positions for effective training
  • Can produce false positives/negatives due to shuttlecock size, background clutter, or lighting changes
  • Needs careful hyperparameter tuning and augmentation strategies to generalize across different match conditions
  • Integration with tracking modules (like ByteTrack/DeepSORT) can be sensitive to detection confidence thresholds

Bottom line: YOLOv8 enables accurate, efficient shuttlecock tracking.
